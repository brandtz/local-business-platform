# UX Screen Reference Index

> **Purpose:** Maps HTML visual reference files (generated by AI UX tool from the UX design brief) to brief screen IDs. Implementing agents should read the corresponding HTML file + brief spec when building Vue components for any screen.
>
> **Source of truth:** [ux-design-brief.md](ux-design-brief.md) remains the authoritative spec for requirements, routes, data shapes, permissions, and module gating. The HTML files provide **layout, interaction, and visual style reference only**.
>
> **Colors:** Ignore all color values in HTML files. All colors are tenant-configurable via `TenantThemeOverride` in `@platform/ui`. Build against semantic design tokens, not hardcoded hex values.

---

## How to Use These References

When implementing a screen:

1. Look up the **Brief ID** (e.g., BA-05) in the table below
2. Read the **HTML file** to understand layout structure, component arrangement, interaction patterns
3. Read the **brief spec** section for that ID to get requirements (routes, data, permissions, module gating, validation)
4. Build Vue components using `@platform/ui` tokens and primitives — do NOT copy Tailwind CDN classes or inline styles from the HTML

### What to Extract from HTML

| Extract | Ignore |
|---------|--------|
| Layout grid (sidebar, content regions, column ratios) | Color hex values |
| Component inventory (tables, cards, modals, panels) | CDN links, external asset URLs |
| Interaction patterns (tab switch, slide panel, modal, drag) | Tailwind utility classes |
| Data shapes (table columns, form fields, card fields) | Hardcoded sample data |
| Responsive breakpoint behavior | Inline `<script>` implementation |

## Shared Component Patterns Across All Portals

These patterns appear consistently across the HTML references and should be implemented as reusable Vue components in `@platform/ui`:

| Component | Observed In | Pattern |
|-----------|-------------|---------|
| **AppShell** (sidebar + header + content) | All admin/platform screens | Fixed sidebar, sticky header, scrollable content |
| **DataTable** | BA-05, BA-08, BA-12, BA-18, PA-06, PA-12 | Sortable columns, filter bar, bulk-select checkboxes, pagination |
| **MetricCard** | BA-01, BA-08, BA-12, PA-01, PA-06, Analytics | Icon, label, large number, trend badge (% + arrow) |
| **StatusBadge** | All screens | Colored pill: Active/Draft/Pending/Completed/Error/etc. |
| **SlidePanel** | BA-05, BA-12 | Right-side slide-in with overlay, header, scrollable content |
| **Modal** | BA-07, BA-17, PA-02, PA-Sign In | Centered overlay with header, form content, footer actions |
| **TabBar** | BA catalog, BA orders, BA customers, BA content, CP menu, CP account | Horizontal tabs with active underline indicator |
| **FormSection** | BA settings, BA content locations, PA config | Grouped fields with labels, validation states |
| **CalendarGrid** | BA-10 | Day/week/month toggle, time-slot rows, staff columns, event cards |
| **RichTextEditor** | BA-14 | Toolbar (bold/italic/lists/link/image), editable content area |
| **SearchToolbar** | BA-05, BA-08, BA-12, CP-02 | Search input + filter button + sort + action buttons |
| **Pagination** | BA-05, BA-08, BA-12, BA-18, CP-02 | Page numbers, prev/next, showing X of Y |
| **StepperWizard** | CP-05 | Multi-step progress indicator with active/completed/pending states |
| **CardGrid** | BA-07 staff, PA-02 tenants, CP-01 categories | Responsive grid of cards with consistent internal layout |
| **ActivityTimeline** | BA-01, BA-18 | Vertical timeline with icon, title, timestamp, description |
| **ToggleSwitch** | BA settings, PA config | Label + description + toggle control |

## Gaps & Consistency Guidance

For screens without a dedicated HTML reference, maintain visual consistency by:

1. **CP-09 (Booking Availability Picker)** — Reuse the stepper pattern from CP-05 checkout. Calendar picker should match BA-10 calendar style. Slot grid should follow the time-grid pattern from the bookings calendar.
2. **CP-19 (Content Pages)** — Simple full-width content area. Match the typography and spacing from the rich text editor output in BA-14.
3. **PA-04 (Tenant Provisioning)** — Multi-step wizard. Reuse stepper pattern from CP-05, form sections from PA config.
4. Any new screens should follow the AppShell + TabBar + DataTable/CardGrid patterns established across the existing references.

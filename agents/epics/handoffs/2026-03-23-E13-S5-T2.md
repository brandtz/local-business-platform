# Handoff: E13-S5-T2 — Admin Dashboard (BA-01)

## Task

Implement the admin dashboard page with KPI cards, revenue chart, recent orders list, activity timeline, and quick action buttons.

## Admin Route Tree with Auth Guards and Module Gating

The dashboard is mounted at `/admin/dashboard` inside the AdminLayout. The route carries `meta.requiresAuth: true` and is gated by the `analytics` module key. If the analytics module is disabled, the sidebar entry is hidden but the route itself still requires auth and returns a "module not available" message.

```
/admin/dashboard   (meta.requiresAuth: true, gatingKey: "analytics")
```

## Sidebar Navigation Data Structure with Gating Keys

```ts
{
  label: "Dashboard",
  icon: "chart-bar",
  route: "/admin/dashboard",
  gatingKey: "analytics"
}
```

## Chart Library Choice and Usage

**No Chart.js dependency.** The revenue chart is rendered as a CSS bar chart using simple `<div>` elements with dynamic `height` styles proportional to the maximum value in the dataset. This keeps the bundle small and avoids a third-party charting dependency.

```ts
// Each bar is a div with:
//   width: (100 / dataPoints.length)%
//   height: (value / maxValue * 100)%
```

## SDK Methods Consumed

- `analytics.dashboard()` — returns KPI values (revenue, orders, customers, conversion rate) consumed by `buildKpiCardDisplay` from `analytics-dashboard-widgets.ts`.
- `orders.list()` — returns recent orders rendered via `buildOrderListRow` from `order-management.ts`.

## Branding Preview Implementation Approach

Not applicable to this task. The dashboard consumes tenant branding passively through the AdminLayout header.

### API Package (`apps/api/src/search/search.service.ts`)
- `SearchService.scoreDocuments()` — computes relevance scores per document:
  - **Exact match**: fieldWeight × (1.0 + exactMatchBoost)
  - **Prefix match**: fieldWeight × (0.8 + prefixMatchBoost)
  - **Substring match**: fieldWeight × 0.5
  - **Freshness bonus**: linear decay over 365 days × freshnessWeight
  - **Score normalization**: capped at 0–1 range
- `SearchService.buildWeightsForEntity()` — auto-generates weights from EntityIndexConfig (primary field gets 1.0, secondary fields get 0.5)
- Default sort when no user sort specified: relevance score descending

## Ranking Strategy
1. **Field weighting**: First searchable field (typically "name") gets full weight, secondary fields get half weight
2. **Match type boosting**: Exact matches score highest, then prefix, then substring
3. **Freshness**: Recent documents get a small bonus that decays linearly over a year
4. **Customization**: Per-entity weights auto-generated from EntityIndexConfig
5. **Override**: User-specified sort parameters override relevance-based ordering

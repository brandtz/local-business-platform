### Implementation Notes

- **Filter bar** includes a date range picker (start/end date inputs) and a text search field. Filters are applied as query parameters to `audit.list()`.
- **Activity table** columns: Timestamp, Actor, Action, Resource, Severity.
- **Severity badges** use color-coded labels derived from `SecurityEventRecord` from `@platform/types`: red for "critical", orange for "warning", blue for "info", gray for "debug".
- **Expandable event details** — clicking a row toggles an inline detail panel showing the full event payload (IP address, user agent, metadata).
- **Pagination** uses a "Load More" button at the bottom of the table. Each call to `audit.list()` returns a cursor; the next page is fetched by passing the cursor to the subsequent call.

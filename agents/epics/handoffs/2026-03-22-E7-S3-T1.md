## Changed Surfaces

### Types Package
- **`platform/packages/types/src/booking.ts`** — New file defining:
  - `LocationHoursInput` — Numeric day-of-week (0=Sun) with open/close times
  - `BookableStaffMember` — Staff member with schedule windows and blackout dates
  - `ExistingBooking` — Existing booking record for conflict detection
  - `AvailabilityInput` — Unified input contract for slot computation (E7-S3-T2)
  - `dayNameToNumber()` / `dayNumberToName()` — Mapping between location-hours string day names and staff schedule numeric day-of-week
- **`platform/packages/types/src/index.ts`** — Added `export * from "./booking"`

### API Service
- **`platform/apps/api/src/booking/availability-input.service.ts`** — New `@Injectable()` service:
  - `assembleAvailabilityInput()` — Assembles all domain records into unified `AvailabilityInput`
  - `convertLocationHours()` — Maps `LocationOperatingHours` (string days) to numeric `LocationHoursInput[]`
  - `assembleBookableStaff()` — Filters active+bookable staff assigned to a service, attaches schedules and blackouts
  - `isDateBlackedOut()` — Checks if a date falls within location blackout windows
  - `getLocationHoursForDay()` — Returns location hours for a specific day-of-week number

## Webhook Event Store Contract
- **Model**: `WebhookEventRecord` — `id`, `provider`, `eventType`, `providerEventId`, `rawPayload` (full JSON string), `status` (pending/processing/processed/failed/skipped), `tenantId`, `connectionId`, `attempts`, `lastError`, `lastProcessedAt`, `createdAt`, `updatedAt`
- **Idempotency key**: `(provider, providerEventId)` — duplicate events return null from `createEvent()`
- **Initial status**: All new events start as `pending`
- **Raw payload preservation**: Exact JSON string as received from provider, stored for audit and replay
- **Timestamps**: `createdAt` set on storage (receipt time), `lastProcessedAt` updated on each processing attempt

## Idempotency Rules
1. Duplicate `(provider, providerEventId)` pair → `createEvent()` returns null
2. Ingress service treats null return as accepted duplicate → no side effects
3. Different providers with same event ID are NOT duplicates (provider scoping)

# E13-S6-T1 Handoff: Category Management (BA-04)

## Date: 2026-03-23

## Component Composition
- `CatalogCategoriesPage` (pages/catalog-categories-page.ts)
- Uses: TabBar (catalog tabs), DataTable (category tree list), SlidePanel (create/edit form), Modal (delete confirmation)
- View-model helpers in `catalog-views.ts`

## SlidePanel Form Fields
| Field | Type | Required | Notes |
|-------|------|----------|-------|
| name | text | Yes | Category name |
| slug | text | No | Auto-generated from name |
| description | textarea | No | Category description |
| parentCategoryId | select | No | Parent category dropdown |
| imageUrl | text | No | Image URL |
| displayOrder | number | No | Sort order |

## Drag-Reorder Implementation
- HTML5 native drag attribute (`draggable="true"`) on table rows
- Full drag-reorder library not added to avoid heavy dependency
- TODO: Implement drag event handlers when a lightweight drag library is approved

## SDK Methods Consumed
| Method | Request | Response |
|--------|---------|----------|
| `catalog.listCategories()` | `CatalogListQuery` | `CatalogListResponse<Category>` |
| `catalog.createCategory()` | `CreateCategoryRequest` | `Category` |
| `catalog.updateCategory()` | `string, UpdateCategoryRequest` | `Category` |
| `catalog.deleteCategory()` | `string` | `void` |

## Category Tree Building
- `buildCategoryTree()` in catalog-views.ts builds nested tree from flat list
- Sorts by `displayOrder` within each level
- Indentation via `depth` property (24px per level)

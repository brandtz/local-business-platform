## Admin Exception-Handling Flows
- **No-Show**: Available when booking is confirmed or checked-in. Admin selects "Mark No-Show" → transitions to no-show terminal state.
- **Late Cancel**: Available when booking is requested or confirmed. Admin provides reason → bypasses cancellation window via adminCancelBooking().
- **Reschedule**: Available when booking is requested or confirmed. Admin selects new time slot → cancels current booking, creates new one. (Creation flow reuses slot computation.)

## Allowed Transitions and Denial Reasons
- Quick actions show only valid forward transitions (cancel is a separate button)
- `canCancel` boolean shown when status is in cancellable set (requested, confirmed)
- Invalid transitions throw BookingTransitionError with explicit denial reason message

## What Changed

### E4-S3-T1: Storefront Layout Regions and Navigation Schema
- New file: `apps/web-customer/src/storefront-layout.ts`
  - `storefrontRegions` — 5 regions: header, hero, sidebar, main-content, footer
  - `StorefrontLayoutSchema` — per-template region visibility, sidebar position, hero toggle
  - `StorefrontNavigationSchema` — per-template navigation items with route paths and module requirements
  - Layout and navigation registries for all 3 vertical templates (restaurant-core, services-core, hybrid-local-business)
  - `getStorefrontLayoutSchema()`, `getStorefrontNavigationSchema()`, `getVisibleRegions()`, `isRegionVisible()`
- New file: `apps/web-customer/src/storefront-layout.test.ts` — 27 tests

### E4-S3-T2: Route-Aware Region Composition
- New file: `apps/web-customer/src/storefront-routes.ts`
  - `RouteRegionConfig` — describes active regions for a given route path
  - `resolveRouteRegionConfig()` — maps route path + layout schema to active regions (home gets hero, order/book suppress sidebar)
  - `buildStorefrontRoutes()` — builds full route table from nav schema with placeholder page components and region config in route meta
  - `buildStorefrontRouteRecords()` — convenience extractor for vue-router
- New file: `apps/web-customer/src/storefront-routes.test.ts` — 30 tests

### E4-S3-T3: Module-Aware Navigation Filtering
- New file: `apps/web-customer/src/storefront-navigation.ts`
  - `filterNavigationByModules()` — filters nav items by tenant's enabled modules
  - `applyModuleGuards()` — attaches `requireTenantModule()` guards to module-gated routes
  - `filterStorefrontRoutes()` / `filterStorefrontRouteRecords()` — combined filter + guard application
- New file: `apps/web-customer/src/storefront-navigation.test.ts` — 18 tests

### E4-S3-T4: Template Composition Interfaces
- New file: `apps/web-customer/src/template-composition.ts`
  - `TemplateCompositionDescriptor` — full layout + navigation + visible regions for a template
  - `composeTemplate()` / `composeAllTemplates()` — build composition descriptors
  - `validateTemplateComposition()` — validates completeness for publish workflows
  - `TemplatePreviewDescriptor` — lightweight preview for onboarding template selection
  - `createTemplatePreview()` / `createAllTemplatePreviews()`
- New file: `apps/web-customer/src/template-composition.test.ts` — 28 tests

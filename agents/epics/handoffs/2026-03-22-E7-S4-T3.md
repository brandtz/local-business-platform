# E7-S4-T3: Customer and Admin API Contracts

## Status: Completed

## Changed Surfaces
- `platform/apps/api/src/booking/booking-api-contracts.ts` — Request validators, admin list query parser, response shape assertions
- `platform/apps/api/src/booking/booking-api-contracts.test.ts` — 20 contract tests

## Dependency Outputs
- **assertValidCreateBookingRequest(payload)** — Validates tenantId, serviceId, staffId, startTime, endTime, durationMinutes
- **parseAdminBookingListQuery(params)** — Parses query string params to AdminBookingListQuery with status validation, date format validation, pagination
- **assertValidCalendarViewQuery(payload)** — Validates tenantId, startDate, endDate
- **Response validators**: assertValidAdminBookingDetailResponse, assertValidAdminBookingListResponse, assertValidCustomerBookingDetailResponse, assertValidCalendarViewResponse, assertValidBookingTrackingResponse
- **BookingApiContractError** — Error class for contract violations

## Customer API Contract
- Create booking: POST with CreateBookingInput
- View booking detail: GET → CustomerBookingDetail
- View booking history: GET → CustomerBookingSummary[]
- Cancel booking: POST with cancellation reason

## Admin API Contract
- List bookings: GET with filters (status, staffId, serviceId, dateFrom, dateTo, search, page, pageSize) → AdminBookingListResponse
- View detail: GET → AdminBookingDetail (includes allowedTransitions)
- Transition status: POST with targetStatus
- Calendar view: GET → CalendarViewResponse
- Pipeline counts: GET → BookingPipelineCounts

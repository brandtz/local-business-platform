## Assumptions and Risks

- Assumptions made:
  - job status is derived from tenant health attention counts since no actual background job infrastructure exists yet; pendingCount and failedCount default to 0
  - audit transitions are counted by presence of `lastLifecycleAuditAt` on summaries, which is a coarse proxy until audit record storage is implemented
  - widget derivation is a pure function on the shared types layer so both backend and frontend can use it
- Risks introduced or still open:
  - when real job infrastructure lands, `PlatformJobStatusWidget` will need to be extended with actual pending/failed counts
  - audit summary is approximate; a proper audit query service will provide more accurate counts

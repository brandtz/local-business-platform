- `WebhookEventSummary` excludes raw payload (listing view)

## Replay from Stored Payloads
1. Platform admin selects a failed event from the list view
2. Admin triggers replay via `validateReplayRequest({ eventId })` → `WebhookEventProcessorService.replayEvent(eventId)`
3. Processor re-parses stored raw payload (NOT re-fetched from provider)
4. All idempotency rules apply — replay of already-processed events is safe
5. Result returned as `WebhookReplayResult` with success flag and new status

## View State Pattern
- Consistent with existing platform-admin views (`PaymentHealthViewState`, `WidgetPanelViewState`)
- Discriminated union: `{ kind: "loading" } | { kind: "ready", ... } | { kind: "error", message }`
- Replay has additional states: `{ kind: "idle" } | { kind: "replaying" } | { kind: "success" } | { kind: "error" }`

## Security Test Evidence
- Summary views do not contain `encryptedCredentials`, `credentialsIv`, `credentialsTag`, `secretKey`, `accessToken`
- Detail views contain only raw webhook payload (provider event JSON), not connection secrets
- `assertNoSecretsInWebhookView()` validates view data for security compliance
- Authorization rejects `tenant` and `anonymous` actor types

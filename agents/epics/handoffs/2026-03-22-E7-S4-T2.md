# E7-S4-T2: Booking Services

## Status: Completed

## Changed Surfaces
- `platform/apps/api/src/booking/booking.service.ts` — BookingService with createBooking, confirmBooking, cancelBooking, checkInBooking, completeBooking, noShowBooking, adminCancelBooking, getAdminBookingDetail, listAdminBookings, getBookingPipelineCounts, getCalendarView, getCustomerBookingDetail, listCustomerBookings, getBookingTrackingData
- `platform/apps/api/src/booking/booking.repository.ts` — BookingRepository with in-memory storage, double-booking prevention guard, calendar block queries, status count aggregation

## Dependency Outputs
- **BookingService** class: Full booking lifecycle management
  - `createBooking(input, existingBookings)` — Validates via SlotComputationService.hasConflict(), then repository-level double-booking guard
  - `confirmBooking(tenantId, bookingId)` — requested → confirmed
  - `cancelBooking(tenantId, bookingId, reason?, policy?, now?)` — Enforces cancellation window policy
  - `adminCancelBooking(tenantId, bookingId, reason?)` — Bypasses window checks
  - `checkInBooking(tenantId, bookingId)` — confirmed → checked-in
  - `completeBooking(tenantId, bookingId)` — checked-in → completed
  - `noShowBooking(tenantId, bookingId)` — confirmed/checked-in → no-show
- **Double-booking prevention**: Two layers — SlotComputationService.hasConflict() for proposed slot validation, BookingRepository.createBooking() for storage-level overlap check (same staff + active status + overlapping time)
- **Error types**: BookingNotFoundError, BookingTransitionError, BookingValidationError
- **DoubleBookingError**: Thrown by repository when overlap detected

## E7-S3 Contracts Consumed
- `SlotComputationService.hasConflict(staffId, proposedStart, proposedEnd, existingBookings)` — Used during booking creation to validate no conflicts
- `ExistingBooking` type — Passed to conflict detection

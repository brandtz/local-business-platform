## Assumptions and Risks

- Assumptions made:

  - recovery codes are best modeled independently from challenge verification so later enrollment and rotation flows stay flexible
- Risks introduced or still open:

  - later MFA API endpoints must not leak whether a user had a valid recovery code or was close to a valid challenge value

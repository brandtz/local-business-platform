## Customer Order Tracking Read Model

### Progress Bar State Mapping
- 5 steps: Order Placed → Confirmed → Preparing → Ready → Completed
- Step states: completed (past), current (active), upcoming (future), skipped (cancelled)
- computeTrackingSteps() maps order status to step states using timestamp presence
- Cancelled orders: steps with timestamps show "completed", rest show "skipped"
- getCurrentTrackingStepIndex(): placed=0, confirmed=1, preparing=2, ready=3, completed=4, cancelled/draft=-1

### Customer Tracking View Model
- OrderTrackingViewModel includes:
  - progressBar: steps with label, state, and timeAgo for each step
  - progressPercent: percentage of completed/current steps (e.g., 60% for preparing)
  - statusMessage: human-readable status text for the customer
  - customerInfo: name, email, phone, fulfillment label, delivery address
  - receipt: itemized line items with modifiers, pricing breakdown, promo code

### Customer vs Admin Read Model Differences
- Customer sees: progress bar, status message, receipt with prices, customer info card
- Customer does NOT see: allowedTransitions, cancellationReason, quick actions, pipeline counts
- Admin sees: all of above plus allowedTransitions, quick actions, cancellation reason, pipeline counts, customer email/phone

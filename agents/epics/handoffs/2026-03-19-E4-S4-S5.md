## Key Decisions

1. **Mount point registry pattern** — Domain modules (E7 commerce, E6 domain) register their components via `registerAccountMountPoint()`. Placeholder pages check the registry at render time and defer to the registered component if available.
2. **Auth guard precedence** — Module guards check auth first, then module enablement. Unauthenticated users always redirect to sign-in regardless of module state.
3. **Tenant-scoped cache keys** — Cache keys include `tenantId` (format: `prefix-tenantId-vVersion`) to prevent cross-tenant cache contamination.
4. **Progressive enhancement** — All device capability and notification APIs degrade gracefully. No API call throws; unsupported environments return typed "unsupported" results.

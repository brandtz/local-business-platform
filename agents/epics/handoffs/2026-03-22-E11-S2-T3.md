# E11-S2-T3 Handoff: Point Accumulation Service

**Task ID:** E11-S2-T3
**Status:** Completed
**Date:** 2026-03-22

## Changed Surfaces

### Types Package (`packages/types/src/loyalty.ts`)
- `PointAccumulationInput`: tenantId, customerId, orderId, orderTotalCents
- `PointAccumulationResult`: pointsEarned, newBalance, newLifetimePoints, previousTier, newTier, tierChanged
- `calculatePointsForOrder()`: pure function — calculates points based on config mode (per_dollar floors to whole dollars, per_order returns fixed amount)

### API Service (`apps/api/src/loyalty/loyalty.service.ts`)
- `LoyaltyService.accumulatePoints(input)`: triggered on order completion
  - Gets or creates program config for tenant
  - Throws `LoyaltyProgramDisabledError` if program not enabled
  - Calculates points via `calculatePointsForOrder()`
  - Credits points to customer account (balance + lifetime)
  - Evaluates tier promotion using `computeLoyaltyTier()`
  - Creates "earn" ledger entry with expiration date (if policy != "never")
  - Returns accumulation result with tier change flag

## Integration Point (E7-S2)
- Order completion triggers `accumulatePoints()` with the order total
- The service is designed to be called from order lifecycle event handlers
- Idempotency: caller should check if points were already awarded for an orderId

## Tier Promotion Logic
- Uses `computeLoyaltyTier()` from customer.ts with tenant config thresholds
- Promotion is evaluated on every accumulation
- Only lifetime points determine tier (not current balance)

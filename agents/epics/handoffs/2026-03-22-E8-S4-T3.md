## What Changed

- Changed surfaces:

  - platform/apps/api/src/notification/notification-delivery.service.ts — adapter implementations and registry
- New or updated contracts:
  - DeliveryAdapter interface: { channel, providerName, send(job), checkStatus() }
  - EmailDeliveryAdapter: stub implementation (providerName: "stub-email"), validates email format
  - SmsDeliveryAdapter: stub implementation (providerName: "stub-sms"), validates phone length
  - InAppDeliveryAdapter: real implementation (providerName: "internal"), persists to in-app store
  - getDeliveryAdapter(channel, store): adapter registry — returns appropriate adapter for channel
  - mapProviderErrorToFailureCategory(errorMessage): maps provider errors to FailureCategory enum
  - isTransientFailure(category): determines if failure is retryable
- Provider-failure taxonomy:
  - "transient": timeout, connection refused, ECONNRESET, 502, 503
  - "rate-limited": rate limit, too many requests, 429
  - "invalid-recipient": invalid email/phone/recipient/address
  - "content-rejected": content/spam/blocked
  - "configuration-error": auth/api key/credential/forbidden/401/403
  - "provider-error": provider/internal server/500
  - "unknown": anything else

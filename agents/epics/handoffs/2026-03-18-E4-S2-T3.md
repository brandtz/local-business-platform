# E4-S2-T3: Backend API Integration for Tenant Config

**Status:** ✅ Complete
**Date:** 2026-03-18
**PR:** Current branch

## Summary

Connected the frontend bootstrap to backend tenant configuration APIs, replacing
the dev-only data source with a real API-backed implementation while preserving
the sessionStorage override as a dev/test fallback.

## Changes

### `apps/web-customer/src/tenant-context.ts`
- Added `createApiBootstrapDataSource(config: ApiClientConfig)` — real
  API-backed data source that calls `GET {baseUrl}/tenant/bootstrap`
- Added `parseBootstrapResponse(body: unknown)` — validates and parses API
  response into `TenantBootstrapData`
- Added `createBootstrapDataSource(config: ApiClientConfig)` — composite data
  source that checks sessionStorage override first, then falls back to API
- Uses `classifyApiError()` from `@platform/sdk` for error classification
- Uses `AbortController` with configurable timeout from `ApiClientConfig`
- Never caches across tenant boundaries — every call is a fresh fetch
- Preserved `createDevBootstrapDataSource()` for existing test compatibility

### `apps/web-customer/src/main.ts`
- Replaced `createDevBootstrapDataSource()` with `createBootstrapDataSource()`
- Added `resolveBootstrapDataSource()` that creates `ApiClientConfig` for
  `"web-customer"` with optional `VITE_API_BASE_URL` env override
- SessionStorage override still works as dev/test fallback

### `apps/web-customer/src/api-bootstrap-data-source.test.ts` (new)
- 31 tests covering:
  - API integration: correct endpoint, request options, response parsing
  - Module payload: enabled modules extraction, empty modules, template key
  - Failure handling: network errors, HTTP errors (401/403/404/500),
    malformed JSON, missing/invalid config → `api-unreachable` shell state
  - SessionStorage fallback: override takes precedence, empty falls to API

## API Contract

The data source expects a `GET /api/tenant/bootstrap` endpoint returning:

```json
{
  "tenants": [TenantResolutionTenantRecord[]],
  "tenantConfig": {
    "templateKey": "restaurant-core",
    "enabledModules": ["catalog", "ordering"]
  }
}
```

## Out of Scope
- Backend API changes (consuming existing endpoints only)
- Offline caching
- Admin app bootstrap

# E11-S2-T4 Handoff: Point Redemption Service

**Task ID:** E11-S2-T4
**Status:** Completed
**Date:** 2026-03-22

## Changed Surfaces

### Types Package (`packages/types/src/loyalty.ts`)
- `PointRedemptionInput`: tenantId, customerId, pointsToRedeem
- `PointRedemptionResult`: pointsRedeemed, discountCents, newBalance
- `computeRedemptionDiscount()`: converts points to discount cents using redemption rate
- `pointsToDiscountCents()`: alternative conversion (floors to whole units)

### API Service (`apps/api/src/loyalty/loyalty.service.ts`)
- `LoyaltyService.redeemPoints(input)`: validates and processes redemption
  - Throws `LoyaltyProgramDisabledError` if program not enabled
  - Throws `LoyaltyValidationError` for zero/negative or below-minimum points
  - Throws `InsufficientPointsError` if balance < requested (**balance never goes negative**)
  - Computes discount using `computeRedemptionDiscount(points, rate)`
  - Debits customer account balance
  - Creates "redeem" ledger entry with negative point delta

## Pricing Engine Integration (E7-S1)
- Existing `LoyaltyCodeService` in `apps/api/src/cart/loyalty-code.service.ts` provides the hook point
- The service's `validate()` method can be extended to call `LoyaltyService.redeemPoints()`
- `LoyaltyCodeService.buildDiscount(valueCents)` creates a `DiscountInput` for the pricing engine
- **Additive integration**: no modifications to E7-S1 core `PricingEngineService` — uses existing discount mechanism

## Balance Constraint
- Point balance MUST never go negative
- Validated before debit: `account.pointBalance < input.pointsToRedeem` → throws
- Minimum redemption enforced: `config.minimumRedemptionPoints`

  - platform/apps/api/src/subscription/subscription-package.service.ts — automatic versioning on entitlement changes, getEntitlementsForVersion, getLatestVersion, listVersions

- Versioning and grandfathering rules:
  1. Every package creation produces version 1 with an entitlementsSnapshot (PackageEntitlementMap as JSON)
  2. When updatePackage changes entitlements (modules, premium features, or usage limits), a new version is created automatically
  3. Price-only or metadata-only changes do NOT create a new version
  4. hasEntitlementChanges(before, after) detects: module adds/removes/toggles, premium feature adds/removes, usage limit value changes/adds/removes
  5. Existing subscribers retain their contracted entitlements at the version they subscribed at
  6. On explicit upgrade or renewal, subscribers move to the latest version
  7. getEntitlementsForVersion(packageId, versionNumber) retrieves any historical version's entitlements

- PackageVersionRecord: { id, createdAt, packageId, versionNumber (auto-incrementing per package), changeReason, changedBy (actorId), changedAt, entitlementsSnapshot (PackageEntitlementMap) }

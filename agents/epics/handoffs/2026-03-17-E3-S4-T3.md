- New or updated contracts:
  - `DomainPromotionWorkflowService` — orchestrates the full promotion lifecycle:
    - `requestReady()`: transitions `not-requested → ready`, `rolled-back → ready`, `failed → ready`; requires `verified` verification state
    - `promote()`: transitions `ready → promoted`; requires `verified` verification state
    - `initiateRollback()`: transitions `promoted → rollback-pending`
    - `completeRollback()`: transitions `rollback-pending → rolled-back`; sets `restoreManagedSubdomain: true`
    - `failPromotion()`: transitions `ready → failed` or `rollback-pending → failed`; records failure reason; `restoreManagedSubdomain: true` when failing from `rollback-pending`
    - `canRequestReady()`: eligibility check for promotion readiness
  - `DomainPromotionWorkflowResult` — discriminated union: `ready`, `promoted`, `rollback-initiated`, `rolled-back`, `failed`, or `ineligible`
  - All promotion operations delegate to `TenantCustomDomainPolicyService.requirePromotionTransition()` for state machine enforcement

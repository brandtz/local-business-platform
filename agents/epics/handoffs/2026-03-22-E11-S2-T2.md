## Changed Surfaces

### Types Package (`packages/types/src/loyalty.ts`)
- `LedgerEntryType`: `earn` | `redeem` | `expire` | `adjust`
- `PointLedgerEntry`: full ledger row with type, points, balanceAfter, referenceType/Id, expiresAt
- `PointLedgerQuery` / `PointLedgerResponse`: paginated ledger query
- Type guard: `isValidLedgerEntryType()`

### Prisma Schema (`apps/api/prisma/schema.prisma`)
- `LedgerEntryTypeEnum`: EARN, REDEEM, EXPIRE, ADJUST
- `CustomerLoyaltyAccount` model: tenantId, customerId (unique pair), currentTier, pointBalance, lifetimePoints, memberSince
- `PointLedgerEntry` model: tenantId, customerId, accountId FK, type, points, balanceAfter, description, referenceType/Id, expiresAt
- Indexes on (tenantId, customerId, createdAt), (tenantId, customerId, type), (expiresAt)

### API Layer (`apps/api/src/loyalty/loyalty.repository.ts`)
- In-memory repository for loyalty accounts and ledger entries
- Methods: getAccount, getOrCreateAccount, updateAccount, listAccountsForTenant
- Methods: addLedgerEntry, getLedgerEntries, getExpirableEntries

## Contracts Introduced
- `PointLedgerEntry` shape — consumed by T3, T4, T5, T6, T7
- `CustomerLoyaltyAccount` (Prisma) — consumed by all loyalty services

## Existing Read Model (from E7-S5)
- `LoyaltyAccount` in customer.ts remains the customer-facing read model
- `LoyaltyPointsHistoryEntry` provides the simplified history view
- The engine types in loyalty.ts provide the full ledger detail

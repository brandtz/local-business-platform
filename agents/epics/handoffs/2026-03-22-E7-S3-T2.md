## Slot Computation Rules
1. **Location hours**: Slots only generated within location open/close times for the requested day
2. **Staff schedule intersection**: Each staff member's schedule windows are intersected with location hours — effective window is the overlap
3. **Blackout windows**: Location blackouts → zero slots for that date. Staff blackouts → zero slots for that staff member on that date
4. **Buffer time**: After each slot, `bufferMinutes` gap before next slot start
5. **Duration**: Each slot spans exactly `durationMinutes` — no partial slots
6. **Lead time (minAdvanceHours)**: When `now` is provided, slots before `now + minAdvanceHours` are excluded
7. **Max advance (maxAdvanceDays)**: When computing date ranges with `now`, dates beyond `now + maxAdvanceDays` are excluded
8. **Conflict detection**: Proposed slots checked against `existingBookings` — overlapping time for same staff member blocks that slot
9. **Multi-staff**: Each staff member generates independent slots; multiple staff increase total available capacity
10. **Deterministic output**: Slots sorted by startTime then staffName for reproducible results

## Conflict Detection Rules
- Full overlap: proposed time exactly matches existing booking
- Partial overlap: proposed start < existing end AND proposed end > existing start
- Adjacent (end == start): NOT a conflict — back-to-back bookings allowed
- Cross-staff: Same time, different staff → no conflict

## Design Decisions

- Shell chrome policy (showNavigation, showFooter) is defined centrally so all 3 apps agree on which chrome regions are visible for each state.
- Loading and auth-required states hide all chrome; ready/error/empty show full chrome; access-denied shows navigation but hides footer; suspended hides all chrome.
- Customer-facing suspended state uses "Store Unavailable" to avoid exposing tenant identity or suspension details to unauthorized visitors.
- HTTP status ‚Üí shell state mapping is intentionally simple (401, 403, 5xx, network) ‚Äî it complements the existing `classifyApiError()` and `shouldTransitionAuthState()` in @platform/sdk without replacing them.

# E13-S4-T5: Loyalty and Rewards

## Summary
Loyalty & Rewards page at `/account/loyalty` with points balance, tier progress bar, tier benefits, available rewards with redeem buttons, and points history table.

## Route
- `/account/loyalty` → `AccountLoyaltyPage`
- Auth guarded via `createAuthGuard()`

## SDK Methods Consumed
- `auth.me()` — get current user ID
- `loyalty.getConfig()` — fetch loyalty program configuration (tiers, redemption rules)
- `loyalty.getCustomerPoints(userId)` — fetch customer loyalty account
- `loyalty.listRewards()` — fetch available rewards
- `loyalty.redeem({ tenantId, customerId, pointsToRedeem })` — redeem a reward

## Components
- `AccountLoyaltyPage` — full loyalty dashboard

## Features
- Large points balance display with tier badge
- Tier progress bar: current tier → next tier with percentage
- "Max tier" message when at highest tier
- Tier benefits list with current tier highlighted
- Available rewards with point cost and "Redeem" button (disabled if insufficient points)
- Points history table: date, description, points (+/-), balance after
- Uses `buildLoyaltyTabData()` from `@platform/types` for canonical data transformation

## Pinia Store Interactions
None — loyalty data is read from API, redemptions go directly to API.

## Empty/Loading/Error States
- Loading: spinner with "Loading loyalty information..."
- Error: alert with "Something went wrong" and "Back to Account" link
- No rewards: "No rewards available at this time."
- No history: "No points activity yet."

- `calculateTierProgress()` — progress calculation (0, halfway, at/over threshold, edge cases)

## Refund Audit Contract
- API validation enforces: `actorId` required, `reason` required, `amountCents` > 0
- Service layer enforces: amount ≤ remaining captured, state must be captured or partially_refunded
- Audit events emitted: `refund_initiated` (before provider call), `refund_completed` (after success)
- All events include: actor, reason, amount, tenant, transaction, provider, status

# E13-S4-T6: Payment Methods

## Summary
Payment Methods page at `/account/payment-methods` with saved cards list, add/remove methods, set default, and confirmation dialogs.

## Route
- `/account/payment-methods` → `AccountPaymentsPage`
- Auth guarded via `createAuthGuard()`

## SDK Methods Consumed
- `payments.listMethods()` — fetch saved payment methods
- `payments.addMethod({ type, token, setDefault })` — add new payment method
- `payments.removeMethod(id)` — remove a payment method
- `transport.patch('/payments/methods/{id}', { isDefault: true })` — set as default

## Components
- `AccountPaymentsPage` — full payment methods management page

## Features
- Saved cards list: brand icon, brand label, last 4 digits, default badge
- "Set as Default" action on non-default cards
- "Remove" action with confirmation modal dialog
- "Add Payment Method" button opens inline form
- Add form: card type select (Visa/Mastercard/Amex/Discover) + token input
- First card auto-set as default
- Empty state: "No Payment Methods" with "Add a payment method for faster checkout"

## Pinia Store Interactions
None — payment method management goes directly to the API.

## Security Notes
- Only gateway tokens are stored — no raw card numbers
- Card token input is a placeholder for payment processor integration
- `PaymentMethodSummary` from SDK only exposes `id`, `type`, `last4`, `isDefault`

## Empty/Loading/Error States
- Loading: spinner with "Loading payment methods..."
- Error: full-page alert when no methods loaded, inline alert when methods exist
- Empty: "No Payment Methods" message with add prompt
- Remove confirmation: modal dialog with confirm/cancel buttons

## Test Coverage
- `getCardBrandIcon()` — brand to icon mapping (known brands, case insensitive, unknown)
- `formatExpiryDate()` — expiry formatting (single digit month, double digit, year truncation)
- 11 tests total

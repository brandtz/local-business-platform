# E13-S4-T2: Profile Settings

## Summary
Profile settings page at `/account/profile` with profile display, edit form, password change with strength indicator, and communication preferences toggles.

## Route
- `/account/profile` → `AccountProfilePage`
- Auth guard: `createAuthGuard()`

## SDK Methods Consumed
- `auth.me()` — fetch current user profile
- `transport.put('/customers/{id}', data)` — update profile (name, phone)
- `transport.post('/auth/change-password', data)` — change password

## Components
- `AccountProfilePage` — full profile settings page
- Uses shared `renderAccountSidebar()` from account-dashboard-page

## Features
- Profile display: avatar initials, name, email
- Edit profile form: display name, phone fields with save button
- Change password form: current password, new password, confirm password
- Password strength indicator: 5 levels (Weak → Very Strong)
- Communication preferences: email promotions, SMS notifications, order updates toggles

## Pinia Store Interactions
None — profile updates go directly to the API.

## Empty/Loading/Error States
- Loading: spinner with "Loading profile..."
- Error: alert with "Unable to load profile" and "Back to Account" link
- Form errors: inline error messages below forms
- Form success: inline success messages after save

## Test Coverage
- `validatePasswordStrength()` — strength scoring (empty, short, various criteria combos)
- `validatePasswordMatch()` — password matching (equal, different, empty)
- 12 tests total

## Webhook Event Store Contract
- **Idempotency key**: `(provider, providerEventId)` ‚Äî composite uniqueness
- **Deduplication**: Repository returns null on duplicate, ingress returns `{ accepted: true, duplicate: true }`
- **Storage before processing**: Events are persisted BEFORE processor is invoked
- **Processing failure tolerance**: Webhook returns accepted even if processing fails (event stored for retry)

## Signature Verification Evidence
- Stripe: `t=<timestamp>,v1=<HMAC-SHA256(timestamp.payload, secret)>` ‚Äî rejects expired (>300s), tampered, wrong secret
- Square: `HMAC-SHA256(notificationUrl+payload, signatureKey)` base64-encoded ‚Äî rejects tampered, wrong key, wrong URL
- Both use constant-time comparison (`timingSafeEqual`) to prevent timing attacks

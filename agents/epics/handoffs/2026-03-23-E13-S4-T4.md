# E13-S4-T4: Booking History and Detail

## Summary
Booking history at `/account/bookings` with upcoming/past sections, and booking detail at `/account/bookings/:bookingId` with full info, timeline, and actions.

## Routes
- `/account/bookings` → `AccountBookingsPage`
- `/account/bookings/:bookingId` → `AccountBookingDetailPage`
- Both auth guarded via `createAuthGuard()`

## SDK Methods Consumed
- `bookings.list({ page, pageSize })` — fetch all bookings
- `bookings.get(bookingId)` — fetch single booking detail
- `bookings.cancel(bookingId)` — cancel an upcoming booking

## Components
- `AccountBookingsPage` — booking list split into Upcoming and Past sections
- `AccountBookingDetailPage` — single booking detail with info, timeline, and actions

## Features
- Upcoming bookings: future dates, non-terminal statuses
- Past bookings: past dates or completed/cancelled/no-show
- Booking cards: service name, date/time, staff, duration, status badge
- Detail info: service, staff, date, time range, duration, status, notes
- Timeline: requested → confirmed → checked-in → completed (with cancelled branch)
- Actions for upcoming: Cancel Booking, Reschedule
- Actions for past: Book Again
- Empty state: "No bookings yet — explore our services" with CTA

## Pinia Store Interactions
None — booking actions go directly to the API.

## Empty/Loading/Error States
- Loading: spinner with "Loading bookings..." / "Loading booking details..."
- Error: alert with message and back navigation
- Empty: "No bookings yet" with "Explore Services" CTA

## Test Coverage
- `formatBookingDate()` — date formatting with weekday (valid, invalid, empty)
- `formatBookingTime()` — time formatting with AM/PM (valid, invalid)
- `isUpcomingBooking()` — future vs past date detection
- `getBookingStatusLabel()` — status label mapping (all 6 statuses, unknown)

# Handoff: E13-S5-T7 — Admin Sign In

## Task

Build the admin sign-in page with email/password login form, remember me checkbox, forgot password flow, and redirect to dashboard on success.

## Admin Route Tree with Auth Guards and Module Gating

The login page is mounted at `/admin/login` **outside** the AdminLayout and does **not** require authentication. Authenticated users who navigate to `/admin/login` are automatically redirected to `/admin/dashboard`.

```
/admin/login   (meta.requiresAuth: false)
```

The navigation guard logic:
1. If `to.meta.requiresAuth` is `true` and user is not authenticated → redirect to `/admin/login`.
2. If `to.path` is `/admin/login` and user is authenticated → redirect to `/admin/dashboard`.

## Sidebar Navigation Data Structure with Gating Keys

The login page does not appear in the sidebar navigation. It is a standalone route rendered without the AdminLayout shell.

## Chart Library Choice and Usage

No chart library is needed for this page.

## SDK Methods Consumed

- `auth.login()` — authenticates the user with email and password credentials. On success, stores the session token and redirects to `/admin/dashboard`.
- `auth.forgotPassword()` — sends a password reset email to the provided address. On success, displays a confirmation message.

## Branding Preview Implementation Approach

Not applicable to this task. The login page uses a minimal branded header (logo + business name) pulled from the tenant's public branding configuration, but no editable preview is involved.

### Implementation Notes

- **Login form** includes email and password fields with client-side validation (required, email format, minimum password length).
- **Remember me** checkbox sets a `persistent` flag on the `auth.login()` call, controlling session duration.
- **Forgot password flow** — clicking "Forgot Password?" toggles the form to a single email input. Submitting calls `auth.forgotPassword()` and shows a success message ("Check your email for reset instructions") without navigating away.
- **Error handling** — invalid credentials display an inline error banner above the form. Network errors show a generic "Unable to connect" message.
- **Redirect** — on successful login, the router pushes to `/admin/dashboard` (or to the originally requested route if a `redirect` query parameter is present).

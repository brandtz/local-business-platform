## What Changed

- Changed surfaces:
  - `apps/web-customer/src/main.ts` — replaced placeholder `mountFailureShell()` with `mountShellState()` that uses `ShellStateDescriptor` and `ShellChromePolicy` from `@platform/ui`. Each bootstrap outcome renders through a dedicated render function: `renderLoadingState()`, `renderErrorState()`, `renderUnresolvedState()`, `renderSuspendedState()`. Error state includes retry CTA for api-unreachable failures. All shell states use `data-shell-state` attributes for testability.
  - New file: `apps/web-customer/src/shell-state.ts` — maps `BootstrapResult` to `ShellState` and `BootstrapShellCategory`. Provides `resolveBootstrapShellPolicy()` for render policies, `describeBootstrapShellState()` for descriptors, and `isRetryableBootstrapFailure()` for retry affordance. Uses `createCustomerShellConfig()` from `@platform/ui` for customer-specific messaging.
  - New file: `apps/web-customer/src/shell-state.test.ts` — 58 tests covering shell state mapping, category classification, render policy resolution, descriptor safety, retry detection, and transition mapping.

- New or updated contracts:
  - `BootstrapShellCategory` — `"loading" | "resolved" | "error" | "unresolved-tenant" | "suspended-tenant"` — higher-level classification of bootstrap outcomes for rendering decisions.
  - `mapBootstrapResultToShellState(result)` — maps `BootstrapResult` to `ShellState`.
  - `classifyBootstrapResult(result)` — maps `BootstrapResult` to `BootstrapShellCategory`.
  - `resolveBootstrapShellPolicy(result)` — resolves full `ShellStateRenderPolicy` for a bootstrap result.
  - `describeBootstrapShellState(result)` — resolves `ShellStateDescriptor` for a bootstrap result.
  - `isRetryableBootstrapFailure(result)` — returns `true` for `api-unreachable` failures.

- New config, env vars, events, or migrations: none

## Assumptions and Risks

- Assumptions made:
  - Unresolved-tenant states (no-host, tenant-not-found) map to "error" ShellState since the ShellState enum does not have a dedicated "not-found" variant. Custom descriptors ("Page Not Found") provide the appropriate UX.
  - Both tenant-suspended and tenant-archived map to the same "suspended" ShellState with the customer shell config's "Store Unavailable" messaging.
  - The retry CTA on api-unreachable simply reloads the page. A more sophisticated retry would be scoped to a future task.

- Risks introduced or still open:
  - Shell state render functions use Vue `h()` directly without `.vue` SFC — consistent with the existing main.ts pattern, but future tasks may want to extract these to SFC components for styling.
  - No CSS is provided for shell state classes (`shell-state`, `shell-state--loading`, etc.) — styling is out of scope for this task and should be addressed when the design system CSS layer is built.

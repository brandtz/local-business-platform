## What Changed

- Changed surfaces:
  - New file: `apps/web-customer/src/tenant-context-consumer.ts` — composables, route guards, module-gating helpers, and immutability enforcement for downstream tenant context consumption.
    - `useTenantContext()` — composable wrapping `TENANT_CONTEXT_KEY` injection with typed return and descriptive error when used outside bootstrap.
    - `useTenantBootstrapResult()` — composable wrapping `TENANT_BOOTSTRAP_RESULT_KEY` injection with typed return and descriptive error.
    - `isTenantModuleEnabled(context, key)` — checks if a single module is enabled.
    - `areAllTenantModulesEnabled(context, keys)` — checks if all specified modules are enabled.
    - `isAnyTenantModuleEnabled(context, keys)` — checks if at least one specified module is enabled.
    - `requireTenantContext(context, options?)` — Vue Router navigation guard that redirects when tenant context is missing.
    - `requireTenantModule(context, key, options?)` — Vue Router navigation guard that redirects when a required module is not enabled.
    - `freezeTenantContext(context)` — deep-freezes tenant context (including enabledModules array) for runtime immutability.
  - Modified: `apps/web-customer/src/main.ts` — now imports `freezeTenantContext()` and freezes the tenant context before providing it to the app. All downstream consumers receive an immutable context object.
  - New file: `apps/web-customer/src/tenant-context-consumer.test.ts` — 32 tests covering composables, module gating, route guards, immutability, and type safety.

- New or updated contracts:
  - `useTenantContext(): TenantFrontendContext` — stable composable API for E4-S3 and E4-S4.
  - `useTenantBootstrapResult(): BootstrapResult` — stable composable API.
  - `isTenantModuleEnabled(context, key): boolean` — stable module-gating API.
  - `requireTenantContext(context, options?): NavigationGuard` — stable route guard API.
  - `requireTenantModule(context, key, options?): NavigationGuard` — stable route guard API.
  - `freezeTenantContext(context): Readonly<TenantFrontendContext>` — immutability enforcement.

## Assumptions and Risks

- Assumptions made:
  - Module-gating helpers accept context explicitly (not via injection) so they can be used in route meta evaluation, setup scripts, and outside component trees. This is more flexible than injection-only access.
  - Route guards accept context as a parameter rather than using injection, since guards execute outside the component setup lifecycle.
  - `Object.freeze()` is sufficient for runtime immutability enforcement — TypeScript's `Readonly<>` provides compile-time safety.

- Risks introduced or still open:
  - If a downstream consumer needs reactive tenant context (e.g., for hot-reload), the frozen context would need to be wrapped in a reactive reference. This is explicitly out of scope per the task definition.
  - Route guards use simple string redirect paths. Future tasks may need more sophisticated redirect logic (e.g., preserving query parameters).

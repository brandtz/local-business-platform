**Date:** 2026-03-23

## Files Created

- `platform/apps/web-customer/src/pages/login-page.test.ts` — Unit tests for password strength evaluator

## Routes
- `/login` → `LoginPage`
- `/register` → `RegisterPage`
- `/forgot-password` → `ForgotPasswordPage`

## SDK Methods Consumed
- `auth.login({ email, password, actorType: "customer", scope: "customer" })` — customer login
- `auth.register({ name, email, password })` — customer registration
- `auth.forgotPassword({ email })` — password reset request

## Login Page Features
1. **Split Layout** — branding panel (left) + form panel (right)
2. **Tab Navigation** — Sign In (active) / Register tabs
3. **Sign In Form** — email, password, remember me checkbox, forgot password link
4. **Post-Login Redirect** — returns to `?redirect=<path>` or home

## Register Page Features
1. **Tab Navigation** — Sign In / Register (active) tabs
2. **Register Form** — name, email, password, confirm password
3. **Password Strength Indicator** — weak/fair/good/strong visual feedback
4. **Client Validation** — password match check, minimum 8 characters

## Forgot Password Features
1. **Email Form** — single email input with submit
2. **Success State** — "Check your email" confirmation with back-to-login link

## Loading/Error State Handling
- Loading: submit button shows "Signing in...", "Creating account...", or "Sending..."
- Error: inline form error message with role="alert"

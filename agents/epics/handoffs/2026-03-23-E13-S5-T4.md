# Handoff: E13-S5-T4 — Payment Gateway Settings (BA-16)

## Task

Implement the payment gateway settings page with provider cards, connection status badges, and a connect form with provider-specific fields.

## Admin Route Tree with Auth Guards and Module Gating

The payment settings page is mounted at `/admin/payments` inside the AdminLayout. It requires authentication and is gated by the `payments` module key.

```
/admin/payments   (meta.requiresAuth: true, gatingKey: "payments")
```

## Sidebar Navigation Data Structure with Gating Keys

```ts
{
  label: "Payment Settings",
  icon: "credit-card",
  route: "/admin/payments",
  gatingKey: "payments"
}
```

## Chart Library Choice and Usage

No chart library is needed for this page.

## SDK Methods Consumed

- `payments.getConfig()` — retrieves the list of available payment providers and their current connection status.
- `payments.createConnection()` — submits a new provider connection using the form data validated by `validateConnectionForm`.

### Implementation Notes

- **Provider cards** are rendered using `buildConnectionListRow` from `payment-connection-management.ts`, which returns display-ready objects with provider name, logo, and status.
- **Connection status badges** use color-coded labels: green for "Connected", yellow for "Pending", gray for "Not Connected".
- **Connect form** fields are dynamically generated by `getProviderFormFields` from `payment-connection-management.ts`, which returns the required field definitions (API key, secret, webhook URL, etc.) per provider.
- **Form validation** is handled by `validateConnectionForm` from `payment-connection-management.ts` before the SDK call is made.

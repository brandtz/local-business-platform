### API Service (`apps/api/src/loyalty/loyalty.service.ts`)
- `LoyaltyService.expirePointsForCustomer(tenantId, customerId)`:
  - Finds all "earn" ledger entries with `expiresAt <= now`
  - Sums expirable points
  - Deducts from customer balance (floors at 0 — never negative)
  - Creates "expire" ledger entry
  - Returns per-customer expiration result
- `LoyaltyService.runExpirationBatch()`:
  - Queries all expirable entries across all tenants
  - Groups by tenant+customer
  - Processes each customer's expiration
  - Returns batch summary

### Repository (`apps/api/src/loyalty/loyalty.repository.ts`)
- `getExpirableEntries(tenantId, beforeDate)`: tenant-scoped expirable entry query
- `getExpirableEntriesAllTenants(beforeDate)`: cross-tenant batch query

## Expiration Policy
- `time_based`: points expire X days after earn date
- `rolling`: same — expiration date set on earn
- `never`: no expiration date set on earn entries
- Expiration date computed at accumulation time: `now + config.expirationDays`

## Orphaned Entry Prevention
- Expiration creates a new "expire" ledger entry to maintain audit trail
- Original "earn" entries are not deleted — they remain with their expiresAt timestamp
- Balance is adjusted atomically with ledger entry creation

## Scheduling
- `runExpirationBatch()` is designed to be called by a cron/scheduled job
- Can be wired to the worker infrastructure (E9-S2 pattern)

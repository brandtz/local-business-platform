# E13-S4-T1: Account Dashboard

## Summary
Account dashboard page at `/account` showing profile card, sidebar navigation, quick stats, and quick links.

## Route
- `/account` → `AccountDashboardPage`
- Auth guard: `createAuthGuard()` redirects unauthenticated users to `/login`

## SDK Methods Consumed
- `auth.me()` — fetch authenticated user profile
- `orders.list({ page: 1, pageSize: 1 })` — get total orders count
- `bookings.list({ page: 1, pageSize: 1 })` — get upcoming bookings count
- `loyalty.getCustomerPoints(userId)` — get loyalty points balance (non-critical, catches errors)

## Components
- `AccountDashboardPage` — main dashboard component
- `AccountSidebar` — reusable sidebar navigation component
- `renderAccountSidebar()` — shared render function used by all account pages

## Sidebar Navigation
- Uses `getAccountNavigationItems()` from `account-routes.ts`
- Highlights current active route
- Mobile dropdown fallback for responsive design

## Pinia Store Interactions
None — dashboard is read-only.

## Empty/Loading/Error States
- Loading: spinner with "Loading your account..."
- Error: alert with message and "Back to Home" link
- Loyalty points default to 0 if API fails (graceful degradation)

## Test Coverage
- `formatMemberSince()` — date formatting (valid, invalid, empty)
- `getQuickStatCards()` — stat card generation (values, links, zero stats)
- 9 tests total

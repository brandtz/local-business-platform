# Handoff: E13-S5-T5 — Tenant User Management (BA-17)

## Task

Build the tenant user management page with tabbed views for members, invitations, and staff. Includes a members table with role badges, action buttons, and an invite modal.

## Admin Route Tree with Auth Guards and Module Gating

The user management page is mounted at `/admin/users` inside the AdminLayout. It requires authentication and is gated by the `users` module key.

```
/admin/users   (meta.requiresAuth: true, gatingKey: "users")
```

```ts

```

## SDK Methods Consumed

- `customers.list()` — retrieves tenant customers shown in the members tab.
- `staff.list()` — retrieves staff members shown in the staff tab.
- `staff.invite()` — sends a staff invitation using the email and role from the invite modal.

### Implementation Notes

- **Tab bar** renders three tabs (Members, Invitations, Staff). The available tabs are determined by `getAvailableTabs` from `user-management-views.ts`, which filters tabs based on the current user's permissions.
- **Permission checks** use `resolveUserSectionPermissions` from `user-management-views.ts` to determine which actions (invite, edit role, remove) the current user may perform.
- **Members table** displays name, email, role badge (Owner / Admin / Member), and action buttons (Edit Role, Remove).
- **Invite modal** contains email and role inputs with client-side validation before calling `staff.invite()`.
- **Role badges** are color-coded: blue for Owner, purple for Admin, gray for Member.

## Changed Surfaces

### API Package (`apps/api/src/search/search-consumer-contracts.ts`)
Document builder functions that transform domain entities into IndexedDocument format:

| Builder Function | Entity Type | Searchable Fields | Facets |
|---|---|---|---|
| `buildCatalogItemDocument()` | catalog-item | name, description | status, visibility, categoryId, price |
| `buildOrderDocument()` | order | id, customerName, customerEmail | status, fulfillmentMode, createdAt, totalCents |
| `buildCustomerDocument()` | customer | displayName, email, phone | createdAt, displayName, email |
| `buildServiceDocument()` | service | name, description | status, isBookable, price |
| `buildBookingDocument()` | booking | customerName, serviceName, staffName | status, staffId, serviceId, startTime |
| `buildStaffDocument()` | staff | displayName, email, role | status, isBookable |
| `buildContentPageDocument()` | content-page | title, slug | status |

- `buildIndexedDocument()` — generic dispatcher that routes to the correct builder by entityType

### API Package (`apps/api/src/search/search.service.ts`)
- `SearchService.search<T>()` — full-text search with filters, sorting, pagination
  - Tenant isolation enforced at repository level
  - Integrates with: admin catalog list, admin order list, admin customer list, storefront product/service list

## Integration Points
Domain services call builders on create/update/delete:
```typescript
// Example: catalog item creation
const doc = buildCatalogItemDocument(item);
searchService.indexDocument(doc);

// Example: order status change
const doc = buildOrderDocument(order);
searchService.handleIndexTrigger("update", doc);

// Example: entity deletion
searchService.handleIndexTrigger("delete", doc);
```

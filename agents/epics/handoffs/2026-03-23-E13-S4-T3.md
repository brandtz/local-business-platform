# E13-S4-T3: Order History and Detail

## Summary
Order history list at `/account/orders` with status filters, pagination, and order detail page at `/account/orders/:orderId` with full receipt and status timeline.

## Routes
- `/account/orders` → `AccountOrdersPage`
- `/account/orders/:orderId` → `AccountOrderDetailPage`
- Both auth guarded via `createAuthGuard()`

## SDK Methods Consumed
- `orders.list({ page, pageSize, status })` — fetch paginated order list
- `orders.get(orderId)` — fetch single order detail

## Components
- `AccountOrdersPage` — order history list with filters and pagination
- `AccountOrderDetailPage` — single order detail with receipt and timeline

## Features
- Status filter tabs: All, Active, Completed, Cancelled
- Order cards: order number, date, total, status badge, item count
- Pagination with previous/next controls
- Detail receipt: items with modifiers, quantities, prices, subtotal, tax, tip, delivery fee, total
- Status timeline using `computeTrackingSteps()` from `@platform/types`
- "Reorder" button linking to menu
- "Back to Orders" navigation
- Empty state: "No orders yet — browse our menu" with CTA

## Pinia Store Interactions
- Reorder button links to `/menu` (cart interaction happens via the menu/item detail flow from E13-S3)

## Empty/Loading/Error States
- Loading: spinner with "Loading orders..." / "Loading order details..."
- Error: alert with message and back navigation
- Empty: "No orders yet" with "Browse Menu" CTA

## Test Coverage
- `formatOrderDate()` — date formatting (valid, invalid, empty)
- `formatCents()` — cents to dollar formatting (0, whole, fractional, large)
- `getOrderStatusLabel()` — status label mapping (all known statuses, unknown)
- `getStatusFilterTabs()` — tab generation (count, keys, labels, filters)

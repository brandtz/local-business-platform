# E4-S6-T1 Task Packet

## Task

- Task ID: E4-S6-T1
- Title: define shared frontend patterns for routing, store setup, API client usage, error boundaries, and auth-state transitions
- Epic: 4 — Shared Frontend System and Customer Experience Shell
- Story: E4-S6 — Cross-App Frontend Conventions
- Owner lane: Frontend Systems Lane

## Scope

Define shared API client and error handling conventions in `@platform/sdk` covering:

- `ApiClientConfig` type and `createApiClientConfig()` factory — per-app config with baseUrl, timeout, credentials
- Error classification: `classifyApiError(status)` maps HTTP status codes to typed `ApiErrorKind` values
- Auth-state transition detection: `shouldTransitionAuthState(error)` returns `"sign-in"` | `"access-denied"` | `null`
- Error boundary pattern: `ErrorBoundaryResult` discriminated union with `caught: true/false` and `recoverable` flag
- Clean factory: `createCleanBoundary()` and `createErrorBoundaryResult(error)`

## Acceptance Criteria

- All types and factories exported from `@platform/sdk`
- Unit tests cover every HTTP error code classification
- Auth-state transition tests cover 401, 403, and other statuses
- Error boundary helpers tested for both clean and caught states
- No runtime HTTP library dependency (conventions only)

# E4-S1-T1 Task Packet

## Task

- Task ID: E4-S1-T1
- Title: define shared visual tokens for typography, spacing, color, elevation, and interaction states
- Epic: 4 — Shared Frontend System and Customer Experience Shell
- Story: E4-S1 — Design Tokens and Layout System
- Owner lane: Frontend Systems Lane

## Scope

Define the design token source of truth in `@platform/ui` covering:

- Typography: font families (sans, mono), font sizes (xs–4xl), weights, line heights
- Spacing: rem-based scale from 0 to 20
- Colors: primitive palette (gray, blue, green, red, amber) and semantic mapping (text, background, border, brand, feedback)
- Elevation: shadow tokens from none through xl
- Border radii: none through full
- Interaction states: focus ring spec, transition speeds, hover/disabled/active modifiers
- Consolidated `DesignTokens` type and `defaultDesignTokens` constant
- `resolveDesignTokens(colorOverrides?)` for tenant-level branding without component forks

## Acceptance Criteria

- All token maps are typed with `as const` and exported from `@platform/ui`
- Unit tests verify token set completeness, semantic color mapping, and override resolution
- No Vue or runtime framework dependency required

- `@platform/utils` (existing — for trimToUndefined)

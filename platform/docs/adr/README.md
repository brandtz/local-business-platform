# Architecture Decision Records

Use this directory for durable decisions that affect platform boundaries or operating rules.

Create an ADR when a change affects:
- architecture boundaries or layering
- multi-tenant isolation assumptions
- CI or release policy
- public package contracts
- cross-cutting operational behavior

Process:
1. copy `0000-template.md`
2. assign the next sequential number
3. write the decision in present tense with concrete consequences
4. link the ADR from the pull request or handoff note that introduced it

Status values:
- proposed
- accepted
- superseded
- rejected
